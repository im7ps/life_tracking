# 📝 Master LangGraph: Question & Answer History

## ✅ Well Answered
- **2026-04-21**: Q: What happens if you remove `Annotated` and `add_messages` from the state? | A: Every message would overwrite the entire conversation.
- **2026-04-21**: Q: What is the specific field in `config` that allows persistence? | A: `thread_id` inside `configurable`.
- **2026-04-21**: Q: Can an Edge modify the state? | A: No, only Nodes can modify the state. Edges define the path.
- **2026-04-21**: Q: What happens if a router returns a string not in the map? | A: It results in a KeyError/Execution error in LangGraph.
- **2026-04-21**: Q: Difference between `ainvoke` and `astream`? | A: `astream` allows sending tokens/events one by one (reducing TTFT), while `ainvoke` waits for the complete response.
- **2026-04-21**: Q: Can I switch between `thread_id="1"` and `thread_id="2"`? | A: Yes, they are physically separated in the DB by their thread ID.
- **2026-04-21**: Q: Why does the AI repeat the tool's output in its final message? | A: It's the agent synthesizing the technical result into a natural response for the user.
- **2026-04-21**: Q: Where should state modifications (like updating a score) happen? | A: In a Node, not a Tool. Tools return data; Nodes apply it to the Graph State.
- **2026-04-21**: Q: How does LangGraph handle parallel nodes return values? | A: Via Accumulation (queuing updates) and Reduction (applying them sequentially using reducers like `add_messages`).
- **2026-04-29**: Q: Explain the lifecycle of an `interrupt` and how the graph resumes. | A: (User Answer: Correct) The graph takes a snapshot, stops, waits for user input, and resumes from that point replacing the interrupt with the input.
- **2026-04-29**: Q: Why inject Rank/Portfolio in the System Prompt? | A: (User Answer: Correct) To reduce latency and "reasoning" overhead, trading token count for speed and direct awareness.

- **2026-04-21**: Q: If a node modifies a global variable, is it saved in the checkpointer? | A: No. The checkpointer ONLY saves the `State`. Global variables are volatile.
- **2026-04-21**: Q: How to handle multiple tools in parallel via Router? | A: (Revised) Return a `list[str]` of node names. LangGraph forks and joins them automatically.
- **2026-04-21**: Q: Parallel vs Sequential DB updates? | A: (Revised) For writes/dependencies, use sequential logic (return one node at a time). For independent reads, parallel is safe.
- **2026-04-29**: Q: Role of `add_messages` reducer. | A: (Revised) It's not just for duplicates, but to prevent total state overwrite. It handles merging by ID (UPSERT logic).
